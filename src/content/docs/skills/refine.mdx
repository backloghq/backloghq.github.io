Reviews the backlog for quality issues and fixes them.

## What it does

1. Identifies vague or unclear task descriptions
2. Finds tasks missing priorities
3. Detects broken or circular dependencies
4. Flags tasks that need specs but don't have them
5. Surfaces stale tasks that may no longer be relevant
6. Applies fixes with your approval

## Usage

```
/backlog:refine
```

Run this periodically to keep the backlog actionable. Especially useful after a planning session to tighten up newly created tasks.
