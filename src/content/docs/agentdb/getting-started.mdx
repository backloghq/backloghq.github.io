## Install

```bash
npm install @backloghq/agentdb
```

## Three Ways to Use It

<Tabs>
<TabItem label="Library">

```typescript
import { AgentDB } from "@backloghq/agentdb";

const db = new AgentDB("./data");
await db.init();

const tasks = await db.collection("tasks");
await tasks.insert({ title: "Hello", status: "active" });

const results = await tasks.find({ filter: { status: "active" } });
console.log(results.records);

await db.close();
```

</TabItem>
<TabItem label="MCP Server (stdio)">

Add to your Claude Code config (`~/.claude/settings.json`):

```json
{
  "mcpServers": {
    "agentdb": {
      "command": "npx",
      "args": ["@backloghq/agentdb", "--path", "./data"]
    }
  }
}
```

Then use `db_find`, `db_insert`, `db_update` etc. from Claude Code.

</TabItem>
<TabItem label="MCP Server (HTTP)">

```bash
npx @backloghq/agentdb --path ./data --http --port 3000
```

Multiple agents connect to the same server with auth:

```bash
npx @backloghq/agentdb --path ./data --http --auth-token secret
```

</TabItem>
</Tabs>

## Quick Example

<Steps>
1. **Insert records**
   ```typescript
   const col = await db.collection("users");
   await col.insert({ name: "Alice", role: "admin" });
   await col.insert({ name: "Bob", role: "user" });
   ```

2. **Query with filters**
   ```typescript
   const admins = await col.find({ filter: { role: "admin" } });
   // → [{ _id: "...", name: "Alice", role: "admin" }]
   ```

3. **Update records**
   ```typescript
   await col.update({ role: "user" }, { $set: { verified: true } });
   ```

4. **Undo mistakes**
   ```typescript
   await col.undo(); // reverses the last mutation
   ```
</Steps>

## Declarative Schemas

Define typed, validated collections with `defineSchema()`:

```typescript
import { AgentDB, defineSchema } from "@backloghq/agentdb";

const db = new AgentDB("./data");
await db.init();

const tasks = await db.collection(defineSchema({
  name: "tasks",
  fields: {
    title: { type: "string", required: true, maxLength: 200 },
    status: { type: "enum", values: ["pending", "active", "done"], default: "pending" },
    priority: { type: "enum", values: ["H", "M", "L"], default: "M" },
    tags: { type: "string[]" },
    due: { type: "date", resolve: (v) => parseDateString(v) },
  },
  indexes: ["status", "priority"],
  tagField: "tags",
  textSearch: true,
  computed: {
    isUrgent: (r) => r.priority === "H" && r.status !== "done",
  },
  virtualFilters: {
    "+OVERDUE": (r) => r.due && new Date(r.due) < new Date(),
  },
  hooks: {
    afterInsert: (id, record, { collection }) => {
      console.log(`Created task ${id}: ${record.title}`);
    },
  },
}));
```

- **Typed fields** — `string`, `number`, `boolean`, `date`, `enum`, `string[]`, `number[]`, `object`, `autoIncrement`
- **Validation** — `required`, `maxLength`, `min`/`max`, `pattern` constraints enforced on insert/upsert
- **Defaults** — static values or functions, applied automatically on insert
- **Field resolve** — transform values before validation (e.g., `"tomorrow"` → ISO date string)
- **Auto-indexing** — B-tree indexes created on collection open
- **Computed fields** — calculated on read, not stored
- **Virtual filters** — domain-specific predicates like `+OVERDUE` usable in queries
- **Lifecycle hooks** — `beforeInsert`, `afterInsert`, `afterUpdate`, `afterDelete` with collection context

## Disk-Backed Storage

For collections that outgrow memory, AgentDB can store data on disk using JSONL and Parquet files:

```typescript
// Disk mode for large collections
const db = new AgentDB("./data", { storageMode: "disk" });

// Auto mode — switches to disk when collection exceeds threshold
const db = new AgentDB("./data", { storageMode: "auto", diskThreshold: 10_000 });

// Per-collection via schema
const events = await db.collection(defineSchema({
  name: "events",
  storageMode: "disk",
  fields: { ... },
}));
```

- **JSONL** for fast point lookups and append writes
- **Parquet** for efficient column scans and analytical queries
- **Lazy index loading** — indexes load on first query, not on open
- **Incremental compaction** — merges small JSONL segments in the background
- **Works on S3** — same disk format with the S3 backend

## Key Features

- **MCP tools** — full CRUD, discovery, schema lifecycle, search, archive, vector search, blob storage, subscriptions
- **Zero native dependencies** — pure TypeScript, no compilation needed
- **Crash-safe** — append-only WAL with immutable snapshots
- **Disk-backed storage** — JSONL + Parquet for collections beyond memory, with auto-switching
- **Undo built-in** — every mutation is reversible
- **Agent identity** — tracks who changed what and why
- **Hybrid search** — BM25 (lexical) + semantic (vector) fused via Reciprocal Rank Fusion
- **Vector search** — HNSW approximate nearest neighbor with 6 embedding providers
- **Unicode tokenizer** — accented Latin (`café`) and CJK (`東京`) indexable out of the box
- **Real-time notifications** — event-driven coordination via `db_subscribe`

## v1.3 → v1.4 Migration

v1.3 included `_id` in the text passed to embedding providers, which made stored embeddings drift from query embeddings and degraded recall. v1.4 fixes the extraction (all `_`-prefixed meta fields excluded). After upgrading, run `reembedAll()` once per embedded collection to rebuild stored vectors with the correct text:

```typescript
const docs = await db.collection("docs");
const result = await docs.reembedAll();
// → { embedded: 1200, failed: 0, errors: [] }
```

Or via MCP: `db_reembed_all { collection: "docs" }` (admin-only, DESTRUCTIVE). See [Search](/agentdb/api/vector-search/) for details.
