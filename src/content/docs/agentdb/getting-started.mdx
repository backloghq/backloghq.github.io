## Install

```bash
npm install @backloghq/agentdb
```

## Three Ways to Use It

<Tabs>
<TabItem label="Library">

```typescript
import { AgentDB } from "@backloghq/agentdb";

// Recommended: one-call factory that constructs and initializes
const db = await AgentDB.open("./data");

const tasks = await db.collection("tasks");
await tasks.insert({ title: "Hello", status: "active" });

const results = await tasks.find({ filter: { status: "active" } });
console.log(results.records);

await db.close();
```

The `new AgentDB(...); await db.init()` two-step still works for backward compatibility, and `db.collection(...)` lazily auto-inits if you skip `init()`. `AgentDB.open()` is the modern recommended path.

</TabItem>
<TabItem label="MCP Server (stdio)">

Add to your Claude Code config (`~/.claude/settings.json`):

```json
{
  "mcpServers": {
    "agentdb": {
      "command": "npx",
      "args": ["@backloghq/agentdb", "--path", "./data"]
    }
  }
}
```

Then use `db_find`, `db_insert`, `db_update` etc. from Claude Code.

</TabItem>
<TabItem label="MCP Server (HTTP)">

```bash
npx @backloghq/agentdb --path ./data --http --port 3000
```

Multiple agents connect to the same server with auth:

```bash
npx @backloghq/agentdb --path ./data --http --auth-token secret
```

</TabItem>
</Tabs>

## Quick Example

<Steps>
1. **Insert records**
   ```typescript
   const col = await db.collection("users");
   await col.insert({ name: "Alice", role: "admin" });
   await col.insert({ name: "Bob", role: "user" });
   ```

2. **Query with filters**
   ```typescript
   const admins = await col.find({ filter: { role: "admin" } });
   // ŌåÆ [{ _id: "...", name: "Alice", role: "admin" }]
   ```

3. **Update records**
   ```typescript
   await col.update({ role: "user" }, { $set: { verified: true } });
   ```

4. **Undo mistakes**
   ```typescript
   await col.undo(); // reverses the last mutation
   ```
</Steps>

## Declarative Schemas

Define typed, validated collections with `defineSchema()`:

```typescript
import { AgentDB, defineSchema } from "@backloghq/agentdb";

const db = await AgentDB.open("./data");

const tasks = await db.collection(defineSchema({
  name: "tasks",
  fields: {
    title: { type: "string", required: true, maxLength: 200 },
    status: { type: "enum", values: ["pending", "active", "done"], default: "pending" },
    priority: { type: "enum", values: ["H", "M", "L"], default: "M" },
    tags: { type: "string[]" },
    due: { type: "date", resolve: (v) => parseDateString(v) },
  },
  indexes: ["status", "priority"],
  tagField: "tags",
  textSearch: true,
  computed: {
    isUrgent: (r) => r.priority === "H" && r.status !== "done",
  },
  virtualFilters: {
    "+OVERDUE": (r) => r.due && new Date(r.due) < new Date(),
  },
  hooks: {
    afterInsert: (id, record, { collection }) => {
      console.log(`Created task ${id}: ${record.title}`);
    },
  },
}));
```

- **Typed fields** ŌĆö `string`, `number`, `boolean`, `date`, `enum`, `string[]`, `number[]`, `object`, `autoIncrement`
- **Validation** ŌĆö `required`, `maxLength`, `min`/`max`, `pattern` constraints enforced on insert/upsert
- **Defaults** ŌĆö static values or functions, applied automatically on insert
- **Field resolve** ŌĆö transform values before validation (e.g., `"tomorrow"` ŌåÆ ISO date string)
- **Auto-indexing** ŌĆö B-tree indexes created on collection open
- **Computed fields** ŌĆö calculated on read, not stored
- **Virtual filters** ŌĆö domain-specific predicates like `+OVERDUE` usable in queries
- **Lifecycle hooks** ŌĆö `beforeInsert`, `afterInsert`, `afterUpdate`, `afterDelete` with collection context

## Disk-Backed Storage

For collections that outgrow memory, AgentDB can store data on disk using JSONL and Parquet files:

```typescript
// Disk mode for large collections
const db = new AgentDB("./data", { storageMode: "disk" });

// Auto mode ŌĆö switches to disk when collection exceeds threshold
const db = new AgentDB("./data", { storageMode: "auto", diskThreshold: 10_000 });

// Per-collection via schema
const events = await db.collection(defineSchema({
  name: "events",
  storageMode: "disk",
  fields: { ... },
}));
```

- **JSONL** for fast point lookups and append writes
- **Parquet** for efficient column scans and analytical queries
- **Lazy index loading** ŌĆö indexes load on first query, not on open
- **Incremental compaction** ŌĆö merges small JSONL segments in the background
- **Works on S3** ŌĆö same disk format with the S3 backend

## Key Features

- **MCP tools** ŌĆö full CRUD, discovery, schema lifecycle, search, archive, vector search, blob storage, subscriptions
- **Zero native dependencies** ŌĆö pure TypeScript, no compilation needed
- **Crash-safe** ŌĆö append-only WAL with immutable snapshots
- **Disk-backed storage** ŌĆö JSONL + Parquet for collections beyond memory, with auto-switching
- **Undo built-in** ŌĆö every mutation is reversible
- **Agent identity** ŌĆö tracks who changed what and why
- **Hybrid search** ŌĆö BM25 (lexical) + semantic (vector) fused via Reciprocal Rank Fusion
- **Vector search** ŌĆö HNSW approximate nearest neighbor with 6 embedding providers
- **Unicode tokenizer** ŌĆö accented Latin (`caf├®`) and CJK (`µØ▒õ║¼`) indexable out of the box
- **Real-time notifications** ŌĆö event-driven coordination via `db_subscribe`

## v2.1 ŌåÆ v2.2 Migration

v2.2 is **additive ŌĆö no breaking changes**. Existing v2.1 code continues to work without changes.

- **Bloom filter query planner integration** ŌĆö `find()`, `count()`, `update()`, and `delete()` now auto-consult any bloom filters on the queried field. Equality predicates (`{ field: value }`, `{ field: { $eq: value } }`, all-absent `{ field: { $in: [...] } }`) short-circuit to zero results when `mightHave` returns false, without touching Parquet or the in-memory store. False positives fall through to scan ŌĆö correct by design. Pre-existing bloom-maintenance bugs (post-creation inserts/updates/deletes were not propagating to bloom) are fixed as part of this work.
- **`HnswOptions.persistEvery`** ŌĆö periodic flush of the HNSW graph sidecar for bounded crash exposure on long-running ingest. Default `undefined` (close-only, v2.1.1 behavior).
- **`HnswOptions.persistTimeoutMs`** ŌĆö bound non-close-time waits on slow backends. Pair with `persistEvery`.
- **Composite + bloom durable persistence** ŌĆö `<dir>/indexes/composite-{fields}.json` and `<dir>/indexes/bloom-{field}.json` mirror the existing B-tree/array persistence. Lazy load on first use; v2.1.1 disk-iter populate retained as fallback. Field-set validation guards against filename collisions.
- **HNSW `seed` propagation** ŌĆö `seed?` now correctly forwards through `CollectionOptions.hnsw` and `AgentDBOptions.hnsw` (latent v2.1.1 omission).
- **Diagnostic warnings** on corrupt JSON / non-ENOENT errors when loading persisted composite or bloom indexes (was silent fall-back).

```typescript
// v2.1 (still works in v2.2)
const db = await AgentDB.open("./data");

// v2.2 ŌĆö periodic HNSW flush for long-running ingest
const db = await AgentDB.open("./data", {
  hnsw: { persistEvery: 1000, persistTimeoutMs: 5000, seed: 42 },
});

// v2.2 ŌĆö bloom filter now consulted by the planner
const users = await db.collection("users");
await users.createBloomFilter("email", 50_000);
await users.find({ filter: { email: "missing@example.com" } });
// ŌåÆ zero records, zero scan (mightHave returned false)
```

**Known issue (carried into v2.2):** the HNSW `graph.bin` sidecar persists to local FS regardless of backend. In S3 deployments with ephemeral container FS, the sidecar is lost on restart and HNSW rebuilds from quantized embeddings (no data loss, just slower cold start). S3-native sidecar is tracked for v2.3 ŌĆö see [Deployment](/agentdb/guides/deployment/#known-issue--hnsw-sidecar-on-s3).

## v2.0 ŌåÆ v2.1 Migration

v2.1 is **additive ŌĆö no breaking changes**. Existing v2.0 code continues to work. The release adds the `AgentDB.open()` factory, lazy auto-init, the configuration pipeline (env vars + `agentdb.config.json`), production-tuning knobs, observability metrics, progress callbacks, and `AbortSignal` support.

```typescript
// v2.0 (still works in v2.1)
const db = new AgentDB("./data");
await db.init();

// v2.1 (recommended)
const db = await AgentDB.open("./data");

// Read live counters without instrumentation
const m = (await db.collection("docs")).metrics();
console.log(m.bm25SegmentCount, m.hnswNodeCount, m.recordCacheHits);
```

v2.1.1 adds HNSW graph persistence (`hnsw/graph.bin` sidecar ŌĆö eliminates O(N) rebuild on reopen for embedded disk-mode collections), deterministic HNSW via `HnswOptions.seed`, and a batch of memory-leak fixes around close/eviction/subscriptions. No code changes required to pick them up.

See [Configuration](/agentdb/guides/configuration/) for the env vars and config-file reference.

## v1.4 ŌåÆ v2.0 Migration

v2.0 replaces the in-house BM25 implementation (`TextIndex`) with [`@backloghq/termlog`](https://github.com/backloghq/termlog) ŌĆö a segment-based LSM full-text index. The legacy `TextIndex`, `TextIndexOpts`, and `IndexFileTooLargeError` exports are removed; the 256 MB on-disk text index cap is gone.

Detection is automatic and fail-loud ŌĆö opening a collection that has a legacy `indexes/text-index.json` blob throws `LegacyTextIndexError`. Recover with a single call:

```typescript
import { AgentDB, LegacyTextIndexError } from "@backloghq/agentdb";

const db = await AgentDB.open("./data");

try {
  await db.collection("docs"); // throws LegacyTextIndexError if v1.4 data is present
} catch (err) {
  if (err instanceof LegacyTextIndexError) {
    await db.rebuildTextIndex("docs"); // builds termlog index, deletes the legacy blob
    await db.collection("docs");        // now succeeds
  }
}
```

Or via MCP: `db_rebuild_text_index { collection: "docs" }` (admin-only, idempotent). The rebuild reads every record once, indexes via termlog, and removes `indexes/text-index.json`.

## v1.3 ŌåÆ v1.4 Migration

v1.3 included `_id` in the text passed to embedding providers, which made stored embeddings drift from query embeddings and degraded recall. v1.4 fixes the extraction (all `_`-prefixed meta fields excluded). After upgrading, run `reembedAll()` once per embedded collection to rebuild stored vectors with the correct text:

```typescript
const docs = await db.collection("docs");
const result = await docs.reembedAll();
// ŌåÆ { embedded: 1200, failed: 0, errors: [] }
```

Or via MCP: `db_reembed_all { collection: "docs" }` (admin-only, DESTRUCTIVE). See [Search](/agentdb/api/vector-search/) for details.
