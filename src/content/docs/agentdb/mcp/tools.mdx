## Tool Reference

### CRUD

| Tool | Description |
|------|-------------|
| `db_insert` | Insert one or many records into a collection |
| `db_find` | Query records with filters, pagination, sorting |
| `db_find_one` | Get a single record by ID |
| `db_update` | Update records matching a filter with `$set`, `$unset`, `$inc`, `$push` |
| `db_upsert` | Insert or update a record by ID |
| `db_delete` | Delete records matching a filter |
| `db_batch` | Execute multiple insert/delete operations atomically |
| `db_count` | Count records matching a filter |

### Discovery

| Tool | Description |
|------|-------------|
| `db_collections` | List all collections with record counts and schema summaries |
| `db_schema` | Sample records to infer field shapes dynamically — works with or without a stored schema |
| `db_distinct` | Get unique values for a field |
| `db_stats` | Database statistics |

### Schema Lifecycle

| Tool | Description |
|------|-------------|
| `db_get_schema` | Read the persisted schema — description, instructions, field types, indexes |
| `db_set_schema` | Create or update a persisted schema (admin-only, overlay merge) |
| `db_delete_schema` | Delete a persisted schema (admin-only, idempotent) |
| `db_diff_schema` | Preview what `db_set_schema` would change — structured diff with warnings and record-impact counts |
| `db_migrate` | Declarative bulk record update via `set`/`unset`/`rename`/`default`/`copy` ops with dryRun and per-record error tracking |
| `db_infer_schema` | Sample existing records and propose a `PersistedSchema` — cold-start schema bootstrap |

### History & Lifecycle

| Tool | Description |
|------|-------------|
| `db_undo` | Reverse the last mutation |
| `db_history` | View mutation history for a record |
| `db_create` | Create a new collection |
| `db_drop` | Soft-delete a collection |
| `db_purge` | Permanently delete a dropped collection |

### Archive

| Tool | Description |
|------|-------------|
| `db_archive` | Move records to cold storage |
| `db_archive_list` | List archive segments |
| `db_archive_load` | Load archived records |

### Search & Vectors

| Tool | Description |
|------|-------------|
| `db_bm25_search` | Lexical BM25 search — no embedding provider required |
| `db_hybrid_search` | BM25 + semantic fused via Reciprocal Rank Fusion |
| `db_semantic_search` | Search by text meaning (requires embedding provider) |
| `db_embed` | Manually trigger embedding for unembedded records |
| `db_reembed_all` | Force-reembed all records (admin-only, DESTRUCTIVE) — v1.3→v1.4 migration |
| `db_vector_upsert` | Store a pre-computed vector with metadata |
| `db_vector_search` | Search by raw vector array |

`db_hybrid_search` and `db_bm25_search` accept `filter`, `limit`, `candidateLimit`, and `summary` params. Hybrid degrades to whichever arm is configured (BM25-only, vector-only, or both); throws only when neither is. See [Search](/agentdb/api/vector-search/).

### Import/Export

| Tool | Description |
|------|-------------|
| `db_export` | Export collections as JSON backup |
| `db_import` | Import from a JSON backup |

### Blob Storage

| Tool | Description |
|------|-------------|
| `db_blob_write` | Attach a file (base64-encoded) to a record |
| `db_blob_read` | Read a blob — returns base64 content + size |
| `db_blob_list` | List blob names for a record |
| `db_blob_delete` | Remove a blob from a record |

### Real-time (HTTP transport only)

| Tool | Description |
|------|-------------|
| `db_subscribe` | Subscribe to collection change notifications |
| `db_unsubscribe` | Stop receiving notifications |

## Response Format

All tools return `{ content: [{ type: "text", text: "..." }] }` with JSON-serialized results. Tools with a declared `outputSchema` also include `structuredContent` for typed programmatic access — clients that know the shape can use it directly instead of parsing the text.

Error responses return `{ isError: true, content: [{ type: "text", text: "error message" }] }` — tools never throw across the MCP boundary.
