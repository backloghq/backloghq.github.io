## Creating a Collection

```typescript
const col = await db.collection("tasks");

// With middleware
const col = await db.collection("tasks", {
  validate: (record) => { if (!record.title) throw new Error("title required"); },
  computed: { wordCount: (r) => (r.title as string)?.split(" ").length },
  virtualFilters: { "+OVERDUE": (r) => (r.dueDate as string) < new Date().toISOString() },
  textSearch: true,
});
```

## Insert

```typescript
// Single record (auto-generates _id)
const id = await col.insert({ title: "Fix bug", status: "open" });

// With agent identity
const id = await col.insert(
  { title: "Fix bug" },
  { agent: "worker-1", reason: "triaged from backlog" }
);

// Batch insert (single disk write)
const ids = await col.insertMany([
  { title: "Task 1" },
  { title: "Task 2" },
]);
```

## Find

```typescript
// All records
const result = await col.find();
// → { records: [...], total: 42, truncated: false }

// JSON filter
await col.find({ filter: { status: "open", priority: { $gte: 3 } } });

// Compact string filter
await col.find({ filter: "status:open priority.gte:3" });

// With pagination, sort, summary mode
await col.find({
  filter: { role: "admin" },
  limit: 10,
  offset: 20,
  sort: "-createdAt",    // descending
  summary: true,         // omit long text fields
  maxTokens: 4000,       // stop when token budget exceeded
});

// Find by ID
const record = await col.findOne("abc-123");
```

## Filtering

Collections accept both **JSON filter objects** (the form shown above) and **compact string filters** like `status:open +bug`. Common shortcuts:

```typescript
// JSON
await tasks.find({ filter: { status: "open", priority: { $gt: 3 } } });
// Compact
await tasks.find({ filter: "status:open priority.gt:3" });
// Tags + text
await tasks.find({ filter: "+bug auth error" });
```

See [Filters](/agentdb/api/filters/) for the full operator reference (including `$strLen`, `$regex`, `$contains`, logical `$and`/`$or`, compact-syntax modifiers, and tag/text shortcuts).

## Update

```typescript
// Update matching records
const count = await col.update(
  { status: "open" },           // filter
  { $set: { status: "closed" } }, // operators
);

// Update operators
await col.update(filter, {
  $set: { status: "closed" },     // set fields
  $unset: { tempFlag: true },     // remove fields
  $inc: { viewCount: 1 },         // increment numbers
  $push: { tags: "reviewed" },    // push to arrays
});

// Optimistic locking
await col.update(
  { _id: "abc" },
  { $set: { status: "claimed" } },
  { expectedVersion: 3 },  // fails if record changed since version 3
);
```

## Upsert

```typescript
const result = await col.upsert("task-1", { title: "Updated", status: "open" });
// → { id: "task-1", action: "inserted" | "updated" }
```

## Upsert Many

```typescript
const results = await col.upsertMany([
  { _id: "task-1", title: "First", status: "open" },
  { _id: "task-2", title: "Second", status: "done" },
]);
// [{ id: "task-1", action: "inserted" }, { id: "task-2", action: "updated" }]
```

Each doc must have an `_id` field. Schema defaults and hooks are applied per document.

## Delete

```typescript
const count = await col.remove({ status: "archived" });
```

## Count

```typescript
await col.count();                           // all records
await col.count({ status: "open" });         // with filter
```

<Aside type="tip">
  When a B-tree index exists on the filtered field and no TTL records exist, `count()` returns the index size directly — O(1) instead of scanning all records.
</Aside>

## Indexes

```typescript
// B-tree index on a scalar field
col.createIndex("status");

// Composite index on multiple fields
col.createCompositeIndex("status", "priority");

// Array index — indexes each element of an array field
col.createArrayIndex("tags");

// Bloom filter for fast non-existence checks
col.createBloomFilter("email");
```

## Undo

```typescript
await col.undo(); // reverses the last mutation
```

## History

```typescript
const ops = await col.history("abc-123");
// → [{ ts, op: "set", id, data, prev, agent, reason }, ...]
```

## TTL (Time-to-Live)

```typescript
await col.insert({ title: "Temp" }, { ttl: 3600 }); // expires in 1 hour
await col.cleanup(); // removes expired records
```

## Dynamic Schema Inspection

> **Note:** this samples actual records to infer field shapes on the fly. For the **persisted schema** (description, instructions, field types committed to disk), see [Schemas](/agentdb/api/schemas/).


```typescript
await col.schema();     // → { fields: [{ name, type, example }], sampleCount }
await col.distinct("status"); // → { field: "status", values: ["open", "closed"], count: 2 }
```

## Blob Storage

Attach files (text or binary) to records. Stored via StorageBackend — works on both filesystem and S3 transparently.

```typescript
// Write
await col.writeBlob(recordId, "spec.md", "# Specification\n...");
await col.writeBlob(recordId, "diagram.png", pngBuffer);

// Read
const content = await col.readBlob(recordId, "spec.md");

// List
const names = await col.listBlobs(recordId);
// → ["spec.md", "diagram.png"]

// Delete
await col.deleteBlob(recordId, "spec.md");
```

<Aside type="note">
  Blobs cascade-delete when the parent record is removed. Blob names must be alphanumeric with dots, hyphens, and underscores.
</Aside>
