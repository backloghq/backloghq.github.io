The MCP CLI (`npx @backloghq/agentdb`) accepts configuration from three sources, merged at startup with a fixed precedence (highest first):

1. **CLI flags** — passed directly to `npx @backloghq/agentdb`
2. **Environment variables** — `AGENTDB_*` prefixed
3. **Config file** — `agentdb.config.json` in the working directory, or the path from `--config` / `AGENTDB_CONFIG`

A field set at a higher layer overrides the same field at a lower layer. Nested objects (e.g. `hnsw`) deep-merge per property — `hnsw.M` from the file plus `hnsw.efSearch` from env both survive into the final config. Available since v2.1.

<Aside type="note">
  Library callers (`new AgentDB(...)` / `await AgentDB.open(...)`) are unaffected by this pipeline — pass options directly to the constructor or factory. The pipeline is also exposed as `loadAgentDBConfig()` for callers who want to reuse it (see [Library API](#library-api)).
</Aside>

## 1. CLI flags

Every option has a flag. Flags win over env vars and the config file:

```bash
npx @backloghq/agentdb \
  --path ./data \
  --http --port 3000 \
  --backend s3 --bucket my-bucket \
  --write-mode group \
  --embeddings openai \
  --schemas "schemas/*.json" \
  --auth-token secret \
  --tenant-id org-123
```

Run `npx @backloghq/agentdb --help` for the full flag list. The CLI exits 1 on invalid numeric flags (e.g. `--port abc`) and on a `--config` path that does not exist.

## 2. Environment variables

All flags have an `AGENTDB_` equivalent. Useful for container deployments and secrets managers.

```bash
AGENTDB_PATH=./data
AGENTDB_WRITE_MODE=group
AGENTDB_BACKEND=s3
AGENTDB_S3_BUCKET=my-bucket
AGENTDB_S3_REGION=us-east-1
AGENTDB_HTTP_AUTH=secret
AGENTDB_TENANT_ID=org-123
AGENTDB_EMBEDDINGS_PROVIDER=openai
AGENTDB_EMBEDDINGS_API_KEY=sk-...
```

<Aside type="caution">
  Environment variables on Linux are readable from `/proc/<pid>/environ` by anyone with access to that file (root or the process owner). For long-lived servers prefer a secrets manager, a read-protected config file (`chmod 600 agentdb.config.json`), or `systemd EnvironmentFile=` rather than exporting tokens directly in shell startup scripts.
</Aside>

Vendor-native API key fallbacks are honoured when `AGENTDB_EMBEDDINGS_API_KEY` is not set: `OPENAI_API_KEY`, `GEMINI_API_KEY`, `VOYAGE_API_KEY`, `COHERE_API_KEY`.

## 3. Config file

`agentdb.config.json` in the working directory is loaded automatically when present. Use `--config <path>` or `AGENTDB_CONFIG=<path>` to point at a different file.

If you pass `--config` explicitly and the file does not exist, the CLI exits 1. The auto-discovered `./agentdb.config.json` silently produces an empty config when absent (no error), so it is safe to omit in development.

```json
{
  "db": {
    "path": "./data",
    "writeMode": "group",
    "maxFindLimit": 5000,
    "memoryBudget": 1073741824,
    "hnsw": { "M": 32, "efSearch": 100 },
    "embeddings": {
      "provider": "openai",
      "model": "text-embedding-3-small"
    }
  },
  "http": {
    "port": 3000,
    "host": "0.0.0.0",
    "auth": "change-me",
    "maxSessions": 200,
    "sessionIdleMs": 600000,
    "cors": ["https://myapp.example.com"]
  },
  "collections": {
    "notes": {
      "maxFindLimit": 500,
      "mergeParquetThreshold": 5,
      "hnsw": { "efSearch": 200 }
    }
  }
}
```

The `collections` key supports per-collection overrides for most storage and search knobs. Any key omitted falls back to the db-wide value, then to built-in defaults. Already-open collections are unaffected by config-file mutations — evict or close the collection for a change to take effect.

`loadAgentDBConfig` emits a `console.warn` when the loaded config file has the world-read bit set (`mode & 0o004` on POSIX) — `chmod 600 agentdb.config.json` to silence it.

## Library API

The config pipeline is also available as a library function. The CLI uses it internally; library users who construct `AgentDB` directly are not affected.

```typescript
import { loadAgentDBConfig, ConfigValidationError } from "@backloghq/agentdb";

try {
  const config = loadAgentDBConfig({
    configPath: "./my-config.json", // optional; auto-discovers agentdb.config.json by default
    requireConfigFile: true,        // error if configPath is missing (default: false)
    env: process.env,               // injectable for testing
    cli: { db: { path: "./data" } }, // highest precedence
  });
  console.log(config.db?.path);
} catch (e) {
  if (e instanceof ConfigValidationError) {
    console.error(`Config error (${e.source}): ${e.message}`);
  }
}
```

`ConfigValidationError` carries:
- `.source` — `"file"` | `"env"` | `"cli"`
- `.path[]` — the field path that failed validation
- `.message` — human-readable, includes the variable name and bad value (raw secrets are redacted, e.g. `AGENTDB_HTTP_MULTI_TOKEN=<redacted> is not valid JSON`)

Returned shape: `{ db?: DbConfig, http?: HttpConfig, collections?: Record<string, CollectionConfig> }`. All three blocks are optional. Type exports: `AgentDBConfigFile`, `DbConfig`, `HttpConfig`, `CollectionConfig`, `LoadConfigOptions`.

## Restart required

Configuration is read once at startup. Changing env vars, the config file, or CLI flags takes effect only after restarting the process. There is no hot-reload.

## Verifying a knob change took effect

After raising or lowering a knob (e.g. `cacheSize`, `filterCacheSize`), restart the process and run representative traffic, then call `col.metrics()` to read live counters:

```typescript
const m = col.metrics();
m.recordCacheHits / m.recordCacheFetches;                                  // disk LRU hit rate
m.filterCacheHits / (m.filterCacheHits + m.filterCompilations);            // filter cache hit rate
m.findTruncations;                                                         // non-zero = hit maxFindLimit
m.bm25SegmentCount; m.bm25NeedsMerge; m.hnswNodeCount; m.parquetRowGroups; // index sizes
m.writeMode;                                                               // open-time write mode
```

All counters are lifetime values since the collection was opened. See [Collection API](/agentdb/api/collection/#observability--colmetrics) for the full snapshot shape.
